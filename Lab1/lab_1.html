Мета роботи

Освоїти фундаментальні операції маніпуляції даними в середовищі R за допомогою сучасного пакета dplyr, використовуючи функціональний підхід та конвеєрне програмування (%>% або |>). Навчитися ефективно фільтрувати, перетворювати, агрегувати та реструктурувати дані для подальшого аналізу чи завантаження в ETL-процеси.

Робоче завдання

Виконати індивідуальне завдання варіанту 3.

Хід роботи

Відфільтрувати діаманти, у яких якість огранки (cut) — "Premium" або "Ideal", а колір (color) — від "D" до "F" (найвища якість). Вибрати лише стовпці: carat, cut, color, clarity, price. Відсортувати за зростанням ціни (price).

Code 
diamonds |>
  filter(cut %in% c("Premium", "Ideal"), color %in% c("D", "E", "F")) |>
  select(carat, cut, color, clarity, price) |>
  arrange(price)

Для кожної комбінації cut (огранка) та clarity (чистота), яка представлена більш ніж у 1000 діамантів, розрахувати:

  • середню ціну (mean_price),
  • медіану ваги (median_carat),
  • кількість діамантів (n).

Використати group_by(cut, clarity), summarise() та across() для агрегації числових стовпців.

Code 
df_diamonds <- diamonds |>
  group_by(cut, clarity) |>
  filter(n() > 1000) |>
  summarise(n = n(),
            across(price, mean, .names = "mean_price"),
            across(carat, median, .names = "median_carat"),
            .groups = "drop")

Взяти результат попереднього пункту.

Перейменувати стовпець cut на Якість_огранки, а clarity — на Чистота. Перемістити стовпець n (кількість) між Якість_огранки та Чистота. Перетворити назви числових стовпців (mean_price, median_carat) на формат з префіксом "Середнє_" та "Медіана_" (наприклад, Середнє_ціна, Медіана_карат).

Code 
df_diamonds |>
  rename(Якість_огранки = cut, Чистота = clarity) |>
  relocate(n, .after = Якість_огранки) |>
  rename_with(~ case_when(
    .x == "mean_price" ~ "Середнє_ціна",
    .x == "median_carat" ~ "Медіана_карат",
    TRUE ~ .x
  ))
