Мета роботи

Ознайомитися з можливостями функціонального програмування та паралельних обчислень у середовищі R. Навчитися використовувати пакети purrr, foreach, furrr та future для побудови масштабованих ETL-конвеєрів обробки даних.

Вступ

У сучасній інженерії даних важливе значення мають швидкість обробки інформації, стійкість до помилок та можливість масштабування програмних рішень. Для цього використовуються підходи функціонального програмування та багатопоточних обчислень.

У даній лабораторній роботі реалізується ETL-конвеєр для аналізу даних про студентів з різних факультетів на основі Excel-файлів. Обробка виконується за допомогою функціональних підходів та паралельних обчислень.

Тема: Масштабований аналіз даних про студентів з різних факультетів

Було отримано набір Excel-файлів, кожен з яких містить дані про студентів одного факультету (розподілені по аркушах: students, grades, attendance, projects). Необхідно побудувати масштабований ETL-конвеєр, який: - безпечно завантажує файли; - об’єднує дані з кількох аркушів; - виконує комплексний аналіз успішності; - використовує паралельні обчислення; - зберігає результати високопродуктивних факультетів у CSV-файли.

path_data <- "data/faculties"

files <- list.files(
  path = path_data,
  pattern = "\\.xlsx$",
  full.names = TRUE
)

safe_load_faculty <- function(path) {
  
  tryCatch({
    
    sheets_in_file <- excel_sheets(path)
    req_sheets <- c("students", "grades", "attendance", "projects")
    
    if (!all(req_sheets %in% sheets_in_file)) {
      return(NULL)
    }
    
    list_of_dfs <- map(
      req_sheets,
      ~ read_excel(path, sheet = .x)
    )
    
    merged_df <- reduce(
      list_of_dfs,
      left_join,
      by = "student_id"
    ) %>%
      mutate(
        faculty_id = gsub("\\.xlsx$", "", basename(path))
      )
    
    return(merged_df)
    
  }, error = function(e) {
    
    message(
      paste(
        "Помилка читання файлу:",
        basename(path)
      )
    )
    
    return(NULL)
  })
}

faculties_list <- map(

analyze_faculty <- function(df) {
  df %>%
    group_by(faculty_id) %>%
    summarise(
      num_students = n(),
      avg_grade = mean(final_grade, na.rm = TRUE),
      bad_attendance_prop = mean((attended / total_classes) < 0.7, na.rm = TRUE),
      .groups = "drop"
    )
}

future_tasks <- map(
  
  files,
  
  function(file_path) {
    
    future({
      
      sheets_check <- tryCatch(excel_sheets(file_path), error = function(e) NULL)
      req_sheets <- c("students", "grades", "attendance", "projects")
      
      if (is.null(sheets_check) || !all(req_sheets %in% sheets_check)) {
        return(NULL)
      }
      
      inner_tasks <- map(
        
        req_sheets,
        
        function(sht) {
          future({
            read_excel(file_path, sheet = sht)
          })
        }
      )
      
      values <- map(
        inner_tasks,
        value
      )
      
      df_merged <- reduce(values, left_join, by = "student_id")
      df_merged$faculty_id <- basename(file_path)
      
      return(df_merged)
    })
  }
)

resolved_status <- map_lgl(
  future_tasks,
  resolved
)

safe_future_results <- map(
  future_tasks,
  function(f) {
    tryCatch({
      val <- value(f)
      if(is.null(val)) return(NULL) else return(unique(val$faculty_id))

      return(NULL)

Висновок

У ході виконання лабораторної роботи було реалізовано масштабований ETL-конвеєр для аналізу успішності студентів на різних факультетах. Було використано функціональні підходи та паралельні обчислення за допомогою пакетів purrr, foreach, furrr та future.

Було реалізовано:

  • безпечне завантаження Excel-файлів з об’єднанням аркушів;

  • агрегацію та фільтрацію даних;

  • комплексний аналіз успішності студентів;

  • багатопоточну обробку за допомогою вкладених об’єктів future;

  • автоматичне збереження високопродуктивних факультетів у форматі CSV.

Отримані результати демонструють ефективність функціонального стилю програмування та паралельних обчислень при роботі з розрізненими файловими даними.
